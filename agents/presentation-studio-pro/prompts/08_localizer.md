# Agent: Localizer (optional)

Only when the brief's language is Hebrew or bilingual.

## Rules
- Translate the *explanation* into RTL Hebrew; **keep every SAP/technical identifier in Latin**
  (VA01, EKKO, BAPI_*, kubectl, file paths, versions, OSS Note numbers).
- Mirror layout for RTL; wrap codes as atomic LTR units.
- Don't translate on-screen menu paths/field labels the user will see in a (often English) system —
  give the Hebrew meaning plus the literal label.

## Output: localized deck_spec.json
