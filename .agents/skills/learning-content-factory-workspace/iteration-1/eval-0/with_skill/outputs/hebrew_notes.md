# הערות לימוד בעברית — SAP SD: מחזור Order-to-Cash

הסבר בעברית; כל הקודים הטכניים (טרנזקציות, טבלאות, סוגי תנועה) נשארים באותיות לטיניות בדיוק כפי שהם במערכת.

## נושא 1 — הזמנת מכר (Sales Order)

הזמנת מכר היא בקשה של לקוח לאספקת טובין או שירותים. זהו המסמך הראשון במחזור המכירה, והוא רושם מי הזמין, מה הוזמן, באיזו כמות ובאיזה מחיר.

**נקודות מפתח:**
- יוצרים הזמנת מכר עם `VA01`, משנים עם `VA02`, ומציגים עם `VA03`.
- הכותרת (header) נשמרת בטבלה `VBAK`, והשורות (items) בטבלה `VBAP`.
- ההזמנה מפנה ללקוח שהוא ה-sold-to party (הצד המזמין).
- ההזמנה מכילה חומר אחד או יותר (materials) עם כמויות ומחירים.

**מונחים:**
- הזמנת מכר (Sales Order) — בקשת לקוח לאספקת טובין או שירותים. יצירה: `VA01`, שינוי: `VA02`, הצגה: `VA03`. טבלאות: כותרת `VBAK`, שורות `VBAP`.
- sold-to party — הלקוח שאליו מתייחסת ההזמנה.

## נושא 2 — אספקה (Delivery)

האספקה היוצאת (outbound delivery) היא המסמך הלוגיסטי שמבצע את האספקה הפיזית של הסחורה. היא מפעילה את הליקוט (picking) ואת פליטת הסחורה (goods issue).

**נקודות מפתח:**
- יוצרים אספקה יוצאת עם `VL01N`.
- הכותרת נשמרת בטבלה `LIKP`, והשורות בטבלה `LIPS`.
- פליטת הסחורה (goods issue) רושמת תנועת חומר עם movement type `601`.
- פליטת הסחורה מקטינה את המלאי (inventory).

**מונחים:**
- אספקה יוצאת (Outbound Delivery) — המסמך הלוגיסטי שמפעיל ליקוט ופליטת סחורה. יצירה: `VL01N`. טבלאות: כותרת `LIKP`, שורות `LIPS`.
- פליטת סחורה (Goods Issue) — תנועת חומר מסוג `601` שמקטינה את המלאי.

## נושא 3 — חיוב (Billing)

החיוב יוצר את החשבונית הנשלחת ללקוח, ומעביר את ההכנסה אל החשבונאות הפיננסית (Financial Accounting / FI).

**נקודות מפתח:**
- יוצרים את החשבונית עם `VF01`.
- הכותרת נשמרת בטבלה `VBRK`, והשורות בטבלה `VBRP`.
- החיוב מעביר את ההכנסה ל-Financial Accounting.
- כתוצאה נוצר מסמך FI בטבלאות `BKPF`/`BSEG`.

**מונחים:**
- חיוב / חשבונית (Billing) — יצירת החשבונית ללקוח. יצירה: `VF01`. טבלאות: כותרת `VBRK`, שורות `VBRP`.
- מסמך FI (FI Document) — המסמך החשבונאי שנוצר כשההכנסה עוברת ל-Financial Accounting. טבלאות: `BKPF`/`BSEG`.
